# skills

这是一个 Claude Code 技能集合，提供自动化问题分发和处理能力。

## 安装

```bash
npx skills add coreylyn/skills
```

安装后，技能会添加到你的 Claude Code 会话中，可以通过斜杠命令调用。

## 包含的技能

### `/autopilot-issues`

自动循环处理 ready-for-agent 的实现问题。持续分发安全的任务批次给子代理，等待草稿 PR/MR，验证合并条件，自动合并通过所有检查的 PR/MR，刷新问题状态，并继续直到没有 ready-for-agent 问题为止。

**适用场景：**
- 需要自动运行问题处理循环
- 自动合并已完成的代理 PR/MR
- 持续分发 ready 问题直到队列清空

**核心特性：**
- 受控循环处理，不是一次性分发
- 严格的合并门控验证
- 自动从草稿转换为 ready-for-review
- 依赖关系和阻塞状态追踪
- 完整的分发账本和最终报告

### `/dispatch-issues`

分析仓库的开放问题、依赖关系、标签、文档和工作树状态，然后仅将当前未阻塞的实现问题分发给子代理。

**适用场景：**
- 需要对 ready 问题进行分类
- 扇出实现任务给多个代理
- 启动独立的问题分支/工作树
- 运行下一批安全的实现工作

**核心特性：**
- 智能就绪性过滤
- 依赖关系映射和阻塞检测
- 自动创建隔离的工作树
- 每个问题一个子代理
- 草稿 PR/MR 创建和验证

## 工作流程

### 基本使用

1. **单次分发：**
   ```
   /dispatch-issues
   ```
   分析并分发当前就绪的问题（最多 3 个）给子代理。

2. **自动驾驶模式：**
   ```
   /autopilot-issues
   ```
   持续循环处理问题，自动合并通过验证的 PR/MR，直到没有就绪问题。

### 工作树管理

所有问题工作树都创建在 `<project-root>/.worktrees/<branch-name>` 目录下，确保每个问题都有隔离的工作环境。

### 支持的平台

- **GitHub**: 优先使用 GitHub connector 工具，否则使用 `gh` CLI
- **GitLab**: 优先使用 GitLab connector 工具，否则使用 `glab` CLI
- **本地 Markdown 问题**: 支持基于文件的问题管理

## 就绪性标准

问题必须满足以下条件才会被分发：

- ✅ 开放且面向实现
- ✅ 具有明确的验收标准
- ✅ 没有标记为需要人工输入或更多信息
- ✅ 所有前置问题已完成
- ✅ 工作可以隔离到独立分支
- ✅ 仓库有足够的文档供代理使用
- ✅ 不需要生产访问或未解决的策略选择

## 合并门控

`autopilot-issues` 仅合并通过所有检查的 PR/MR：

- PR/MR 由子代理创建
- 关联到正确的问题
- 从草稿转换为 ready-for-review
- 差异限制在验收标准内
- 必需检查通过
- 测试已运行并通过
- 无未解决的审查评论或冲突
- 问题无新的阻塞因素
